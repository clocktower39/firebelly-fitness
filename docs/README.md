# Firebelly Docs

This directory is the durable project memory for humans and coding agents.

- `PROJECT_CONTEXT.md`: what the product is and who uses it.
- `ARCHITECTURE.md`: how the client, server, database, auth, and realtime layers fit together.
- `DECISIONS.md`: architecture decisions that should survive across sessions.
- `TODO.md`: prioritized work that is not done yet.
- `FEEDBACK.md`: review notes, risks, and quality observations.
- `SESSION_HANDOFF.md`: current verified state and next best action.
- `CLEAN_STATE_CHECKLIST.md`: checklist before ending a session.
- `EVALUATOR_RUBRIC.md`: rubric for reviewing agent output.
- `QUALITY.md`: codebase quality snapshot.
- `program-builder.md`: program builder domain notes.
